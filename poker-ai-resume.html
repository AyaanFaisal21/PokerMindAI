POKERMIND
AI

Multi-Agent Swarm System // Behavioral Evasion Research // AWS Deployment
Technical Description

PokerMind AI is a distributed multi-agent system designed to autonomously participate in online poker environments through browser automation, while employing a council-of-agents architecture to model optimal decision-making, behavioral consistency, and human-like play patterns simultaneously.

The system's core innovation is a swarm intelligence orchestration layer in which four specialized LLM-backed agents engage in structured inter-agent debate before each game action is committed. A Strategist Agent computes GTO-aligned ranges and pot equity; a Historian Agent maintains a live statistical model of session behavior to detect and correct drift; a Humanizer Agent samples from learned behavioral distributions to introduce timing variance, simulated mistakes, and fatigue patterns; and a central Orchestrator Agent synthesizes weighted recommendations from all three into a final action.

The evasion subsystem — the project's primary technical contribution — models the statistical fingerprint of human poker play using historical hand data, training a behavioral prior that the Humanizer Agent samples from at inference time. This directly addresses detection vectors including action-timing regularity, decision-tree periodicity, bet-sizing quantization, and session-length anomalies — the primary signals used by platform bot-detection heuristics.

The entire system is containerized with Docker and deployed to AWS ECS, with each agent running as an isolated microservice behind an internal message broker. A Playwright-based browser automation layer handles real-time table interaction, while a PostgreSQL session store feeds the Historian Agent's inference context. A live observability dashboard exposes agent reasoning traces, decision confidence scores, and behavioral drift metrics in real time.

Agent Architecture
Strategist
GTO Reasoning Engine
Computes game-theoretically optimal action distributions using pot odds, implied odds, and opponent range modeling. Outputs a probability-weighted action recommendation and confidence score to the Orchestrator.
Historian
Statistical Session Monitor
Maintains a rolling statistical model of session behavior — VPIP, PFR, aggression frequency, timing distributions — and flags drift from established human baselines, sending corrective signals upstream.
Humanizer
Behavioral Distribution Sampler
Samples from learned priors of human poker behavior to inject realistic timing jitter, deliberate suboptimal decisions, tilt simulation, and break patterns. Primary evasion subsystem.
Orchestrator
Decision Synthesis Layer
Ingests structured outputs from the three specialist agents, applies a learned weighting scheme, and commits the final table action via the Playwright automation layer. Maintains full reasoning trace for auditability.

03
Containerized a resource-intensive multi-agent system using Docker + AWS ECS/ECR, deploying each agent as an isolated microservice with Redis message brokering; configured CloudWatch alarms and auto-scaling policies to maintain sub-200ms inter-agent latency under load
04
Built a browser automation layer using Playwright for real-time table interaction, integrating with a PostgreSQL session store feeding live statistical context to the Historian Agent; system maintains a rolling behavioral model across VPIP, PFR, aggression frequency, and timing distributions
05
Implemented a real-time observability dashboard in Grafana exposing agent reasoning traces, per-agent confidence scores, and behavioral drift metrics; enabled live debugging of multi-agent consensus failures and action latency spikes during concurrent sessions
06
